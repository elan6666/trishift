1.3 当前默认配置快照（以代码和 defaults.yaml 为准）
• 全局：matching_mode=ot, k_topk=300, train_mode=joint, n_eval_ensemble=300, seed=24。
• Stage1 ：epochs=100, batch_size=256, lr=5e-4, beta=1.0, z_dim=128。
• Stage2 ：predict_delta=true, shift_input_source=state, use_cross_attention=true,
use_transformer_block=false。
• Stage3 ：input_mode=full, state_dim=cond_dim, use_residual_head=false。
• 损失：gamma=0, lambda_dir_expr=0.01, lambda_dir_z=0.05, lambda_z=1, lambda_expr_mse=0。
• 消融相关：topk_strategy=random, latent_loss_type=gears, per_condition_ot=false。
1.4 模型输入输出速查
Stage1（V AE） 输入： 表达向量x ∈ RG。 输出： 潜均值zµ ∈ Rz_dim， 并缓存到adata.obsm["z_mu"]。
训练监控：有 val 集时以 val_loss 早停并回滚到 best 权重。

Stage2（Shift） 条件向量 c ∈ Rdc 由扰动基因 embedding 聚合得到（sum/mean） 。
• shift_input_source=latent_mu：shift 输入是 zµ
ctrl。
• shift_input_source=state：shift 输入是 Stage3 compressor 的 zstate。
当开启 cross-attention 或 transformer 时， 若dc ̸= dmodel， 会先做线性投影； 若相等则是Identity。
输出维度：
• predict_delta=true：输出 c∆z ∈ Rz_dim；
• predict_delta=false：输出 shift 表征（维度为 model_dim 或 transformer concat 维
度） 。
Stage3（Generator） 输入：控制表达xctrl、条件向量c、shift 输出 h。input_mode 决定拼
接方式：
• full: [c, zstate, h]
• state_fusion: [zstate, h]
• fusion_only: [h]
输出：预测扰动表达 ˆx。
1.5 训练与切分的关键行为
• T rain/V al控制细胞不重叠：切分时显式做 ctrl disjoint，避免 train/val ctrl 泄漏。
• Stage1 可用 val loss：当前实现支持 Stage1 val loss，并用其做早停监控。
• OT 家族默认不复用缓存：仅 knn 默认使用 top-k 缓存，ot/soft_ot/knn_ot 默认重算。
• OT 已可返回权重：当策略需要加权（如weighted/weighted_sample）时，ot 分支会返
回归一化权重。
• loss_z 开关是派生的：predict_delta=false 时会自动关闭 latent 监督。
• stage3_only 限制：train_mode=stage3_only 仅支持model.stage3.input_mode=full。
1.6 与 scPRAM / scouter / systema 的关系
• scPRAM：Stage1 使用 scPRAM 风格 V AE与相关评估指标。
• scouter：condition split 逻辑（尤其Norman subgroup）与输出格式对齐，支持直接对比
metrics.csv 与 mean_pearson.txt。
• systema：集成基于参考向量的 Pearson delta 相关指标，用于方向一致性评估。

1.7 新会话建议工作流
1. 先看 configs/defaults.yaml，确认本次实验的真实开关。
2. 再看 scripts/run_dataset.py，确认配置如何透传到模型和训练函数。
3. 然后看 src/trishift/_model.py，核对当前输入输出维度路径。
4. 最后看 src/trishift/TriShift.py 与 TriShiftData.py，核对 split、top-k、loss 和
early-stop 细节。
1.8 一句话总结
T riShift是在 scouter 流程骨架上，融合 scPRAM 的潜空间学习和 systema 风格评估，并
通过可切换的 Shift–Generator 结构（MLP/Cross-Attention/T ransformer）实现可消融的扰动
表达建模。
